서버 사이드 렌더링(Server-Side rendering, SSR)
- 전통적인 MVC 웹 애플리케이션
- Node.js EJS 뷰 엔진을 쓰는 방식
- 동작 방식
	1. 브라우저가 요청을 보냄
	2. 서버가 DB에서 정보를 조회
	3. EJS 템플릿 + 데이터를 합쳐서 완성된 HTML을 만듦
	4. 완성된 HTML을 브라우저로 전송
	5. 브라우저는 HTML 그려서 화면에 보여줌
- 장점
	- 구조 단순
	- 빠르게 페이지 만들기 좋음
클라이언트 사이드 렌더링(CSR, SPA)
- 프론트엔드와 백엔드를 분리한 아키텍처
- API 서버 + 프론트(순수 JS, React/Vue) 분리
- 동작 방식
	1. js 코드 안에서 fetch 또는 axios 등으로 Node.js 서버의 REST API를 호출
	2. 프론트에서 JSON 데이터를 받아서 DOM을 직접 만들거나 React/Vue 컴포넌트로 렌더링
- 장점
	- 프론트와 백엔드가 역할이 명확하게 분리
	- API 재사용성이 높음(모바일 앱, 웹 서비스, 외부 공개 API 등으로 확장하기 좋음)
	- 프론트 프레임워크(React / Vue 등)와 연동이 좋음
